# Security Policy

## Supported versions

Security fixes are prioritized for the latest tagged release and the current `main` branch.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| `main` | Yes |
| Older releases | Best effort only |

If you are running an older version, please reproduce the issue against the latest release before reporting when possible.

## Reporting a vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Instead, report security issues privately to:

- `Anselm.Hahn@gmail.com`

Include as much detail as you can:

- affected version or commit
- impact and attack scenario
- reproduction steps or proof of concept
- any relevant logs, screenshots, or configuration details

Reports will be reviewed privately and coordinated fixes will be prepared before public disclosure when practical.

## Disclosure expectations

Please give maintainers a reasonable opportunity to investigate and remediate the issue before sharing details publicly.

Once a fix is available, release notes and repository history will document the change as appropriate.
